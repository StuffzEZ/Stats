📄 txtdb

A plain-text stats database for your GitHub profile. Auto-updated, community-verified.

Each Stats user gets a db/USERNAME.txtdb file in this repo. It's auto-generated by your workflow, validated by GitHub Actions, and readable by the stats dashboard. You control exactly what's public.

How it works

1

Your workflow runs (every 6h)

It fetches your stats, updates your README, then writes a fresh username.txtdb file locally.

2

It opens a PR to this repo

The workflow opens a pull request to StuffzEZ/Stats targeting db/YOUR_USERNAME.txtdb.

3

PR is auto-validated

The validate-txtdb-pr workflow checks: is the PR author editing only their own file? Is the username inside the file correct?

4

Auto-merged if valid, auto-closed if not

Valid PRs are squash-merged automatically. Invalid ones (wrong file, wrong username) are closed with an explanation comment.

5

Stats dashboard reads the .txtdb

When anyone visits your stats page, it fetches your .txtdb to supplement public API data with anything you've opted into sharing.

File format

The .txtdb format is an INI-style plain text file with section headers. It's designed to be human-readable and easy to parse.

# txtdb for myusername
# Auto-generated — safe to commit, safe to read publicly

[meta]
username          = myusername
last_updated      = 2026-03-21T13:00:00Z
workflow_version  = 2.0.0
stats_page        = https://stuffzez.github.io/Stats/stats.html?user=myusername

[public_stats]
public_repos      = 24
followers         = 142
total_stars       = 891
public_commits_est= 1204
account_created   = 2020-01-15
member_since_days = 2257

[opted_in]
show_private_repo_count        = false
show_all_orgs                  = true   # ← shows private org memberships
show_private_commit_count      = false
show_custom_notes              = true
show_pinned_override           = true

[private_stats]
# Only populated if opted_in above is true
# private_repos = (not opted in)

[orgs]
org_1_login  = my-company
org_1_name   = My Company
org_1_avatar = https://avatars.githubusercontent.com/u/12345678
org_1_public = false  # private membership, shown because show_all_orgs=true
org_1_visible= true

[pinned_projects]
pin_1 = my-best-project
pin_2 = another-cool-repo

[custom]
website     = https://mysite.dev
pronouns    = they/them
hire_status = open_to_work
note        = Building things at night 🌙

Sections

[meta]
Auto-generated. Username, last update timestamp, workflow version, stats page URL.
[public_stats]
Auto-generated from public GitHub API. Repos, stars, followers, commits, member since.
[opted_in]
Your privacy controls. Five toggles: private repos, all orgs, private commits, custom notes, pinned override. Edit freely.
[private_stats]
Populated when opted in: private repo count, total commits including private repos.
[orgs]
Public orgs by default. Set show_all_orgs=true to include private org memberships — their names and avatars are then public via this file.
[pinned_projects]
Set show_pinned_override=true and list repo names in order. Overrides the default "sorted by last push" on your stats page.
[custom]
Freeform key-value pairs: website, pronouns, hire status, custom note. All shown on your stats page.

Privacy model

Everything in your .txtdb is publicly readable — it's a file in a public repo. Only put things in it that you're comfortable being public. The opt-in flags simply control whether your workflow writes that data into the file.

Your PAT is never written to the .txtdb file. The workflow only reads your private data (orgs, private repos) to count it and optionally include the count — not the content of private repos.

PR validation rules

Every PR to this repo is checked by the validate-txtdb-pr GitHub Actions workflow. A PR is automatically closed if:

  • ✗ It touches any file outside db/
  • ✗ The file being modified is not db/PR_AUTHOR_USERNAME.txtdb
  • ✗ The username field inside the file doesn't match the PR author

A PR is auto-merged if all three checks pass. The author receives a comment with links to their stats page and their .txtdb file.

Manual editing

You can manually edit your .txtdb file's [opted_in], [pinned_projects], and [custom] sections directly on GitHub. The next time your workflow runs, it will preserve your opt-in settings but refresh the [meta], [public_stats], and [orgs] sections with fresh data.

⚠️ Don't manually edit the [meta] or [public_stats] sections — they'll be overwritten on the next workflow run. Only edit [opted_in], [pinned_projects], and [custom].
